# Deploy on Render

This project has two parts:

- `backend/` deploys on Render
- `frontend/` remains an Android app and must be rebuilt with your deployed API URL

## 1. Create the backend service

1. Open [Render Dashboard](https://dashboard.render.com/)
2. Click `New +` -> `Blueprint`
3. Connect this GitHub repo:
   - `Kandpalharsha/Whatsapp-AI-reply-assistant`
4. Render will detect the root [`render.yaml`](/C:/Users/harsh/Downloads/app/local-deploy/render.yaml)
5. Continue with the setup

## 2. Add required secret environment variables

In Render, set these secret values when prompted:

- `OPENAI_API_KEY`
- `MONGODB_URL`

Recommended:
- Use a MongoDB Atlas connection string for `MONGODB_URL`
- Keep `JWT_SECRET` generated by Render

## 3. Understand FAISS storage behavior

The backend stores trained vector indices in `backend/faiss_indices/`.

On Render free instances, the filesystem is ephemeral. That means:

- trained chat indices can be lost after redeploy/restart
- you may need to retrain chats after a fresh deploy

If you want training data to survive redeploys:

- use a paid Render instance with a persistent disk, or
- move the FAISS data to a separate persistent storage design later

## 4. After deploy succeeds

Your backend URL will look like:

`https://your-service-name.onrender.com`

Test:

- `https://your-service-name.onrender.com/health`
- `https://your-service-name.onrender.com/docs`

## 5. Update the Android app to use the deployed backend

Edit [api.ts](/C:/Users/harsh/Downloads/app/local-deploy/frontend/src/services/api.ts) and replace the current local URL with your Render URL, for example:

```ts
export const BASE_URL = 'https://your-service-name.onrender.com';
```

Then rebuild the Android app:

```powershell
cd frontend
npx react-native run-android --port 8082
```

## 6. Important note

Render only deploys the backend API here.

The React Native app is not hosted on Render like a website. It must still be:

- run locally for testing, or
- built into an APK/AAB for Android distribution
